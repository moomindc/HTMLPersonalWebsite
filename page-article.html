AI agent frameworks —
six months in,
what actually matters

I've been testing OpenClaw, Zapier Agents, and Claude in M365 since late 2025. Here's what survived contact with real work in an O365 environment.

The setup

I work in an O365 environment and spend a non-trivial amount of time on inbox management, cross-source research, and scripting repetitive tasks. Agent frameworks promised to help with all three. I was sceptical — but curious enough to spend six months finding out.

"The gap between an agent demo and an agent that survives a Monday morning inbox is larger than any vendor will tell you."

What I tested

Three frameworks got a real workload over six months: OpenClaw for autonomous task running, Zapier Agents for M365 integration, and Claude's built-in tools via the API. Each was tested against actual work — not toy examples or synthetic benchmarks.

frameworks = ["OpenClaw", "Zapier", "Claude API"]
for f in frameworks:
    score = test_against_real_work(f)
    print(f"{f}: {score}/10")

What actually works

Inbox monitoring with clear rules: all three handle this adequately. The differentiator is how gracefully they fail on ambiguous cases. Claude via API is the most honest about uncertainty. OpenClaw is the most autonomous but requires the most upfront setup investment.

Cross-source research is where the gaps show most clearly. Tasks that require pulling from more than two sources with conditional logic are still brittle in all three. OpenClaw handles it best when the workflow is pre-defined. Claude via API handles it best when it isn't.

The verdict

If you're in M365 and want the lowest-friction entry point: Claude in M365 or Zapier Agents. If you want the most capable autonomous runner and are willing to invest in setup: OpenClaw. If you want to build custom: Claude API, no question.

None of them are ready to run unsupervised on anything that matters. That's not a criticism — it's just where the technology is right now. Build in checkpoints.
